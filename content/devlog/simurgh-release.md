Today I've just about wrapped up development of project "Simurgh". A detailed description can be found here: /projects/simurgh. 

Overall I'm very surprised at how well it worked, I though the complexity of the problem would be too difficult and the model would collapse, but I'm happy to see it succeed. Even more so, I feel like the lessons I learned with taking my time with the data prep payed off. The time I spent on the model itself was minimal in comparison, and I feel like that's a good sign. After enormous prep, it was like the cherry on top. 

Every mistake I made during my thesis was "adressed" so to say, there were no major setback, just slow and calculated progress. I do think however I should have taken even more time to fully read the background papers, having the end in sight I rushed through them, looking just for the kernel of the idea. For next time I definitely need to curb this excitement and take even more time, understanding th architectures and solutions I'm using in even greater detail. The documentation itself could also use some work? It's very hard to choose just how in-depth to go, one hand it would be great to write EVERYTHING, but this makes it into a huge amount of work (writing these posts and readme's is already time-consuming enough). On the other - all the details are in the code itself, free to read. Ideally I would want each description to be thesis-level detailed, but we will see just how close to that I can consistently get.

In any case, this project was pretty fun to do, even though the topic was quite niche and "boring". The clear results, easy and standard data formats along with the low file sizes made it enjoyable.

Looking forward to the next one!
